# <a name="motivations-and-principles"></a><span data-ttu-id="e5a67-102">Мотивы и принципы</span><span class="sxs-lookup"><span data-stu-id="e5a67-102">Motivations and Principles</span></span>

## <a name="motivations-behind-the-format"></a><span data-ttu-id="e5a67-104">Мотивы, определяющие формат</span><span class="sxs-lookup"><span data-stu-id="e5a67-104">Motivations behind the format</span></span>

<span data-ttu-id="e5a67-105">В начале 2016 года несколько команд в корпорации Майкрософт (включая команды по разработке Outlook, Windows и Bot Framework) приступили к реализации необходимых им решений ("карточек"). Они были очень похожи, и все команды разрабатывали свои решения независимо друг от друга.</span><span class="sxs-lookup"><span data-stu-id="e5a67-105">In early 2016, several teams at Microsoft (including Outlook, Windows and the Bot Framework) came to the realization that they all wanted something extremely similar ("cards") and that each of them were designing their own solutions independently:</span></span>

- <span data-ttu-id="e5a67-106">В Windows были собственные живые плитки и формат уведомлений.</span><span class="sxs-lookup"><span data-stu-id="e5a67-106">Windows had its own Live Tiles and Notifications format</span></span>
-  <span data-ttu-id="e5a67-107">В Bot Framework использовался набор стандартных шаблонов карточек, которые разработчики могли выбирать при отправке сообщений Bot.</span><span class="sxs-lookup"><span data-stu-id="e5a67-107">The Bot framework was using a set of predefined card templates developers could choose from when sending Bot messages</span></span>
- <span data-ttu-id="e5a67-108">В Outlook использовался собственный формат MessageCard для сообщений с действиями.</span><span class="sxs-lookup"><span data-stu-id="e5a67-108">Outlook was using its own MessageCard format for its Actionable Messages feature</span></span>

<span data-ttu-id="e5a67-109">В то же время другие платформы, такие как LINE, FaceBook Messenger, Slack и прочие, определяли собственный формат карточки.</span><span class="sxs-lookup"><span data-stu-id="e5a67-109">At the same time, other platforms such as LINE, FaceBook Messenger, Slack and more were defining their own, proprietary "card" format.</span></span> <span data-ttu-id="e5a67-110">Так что несколько сотрудников корпорации Майкрософт собрались и начали работать над определением единого открытого формата карточки и набора пакетов SDK, которые:</span><span class="sxs-lookup"><span data-stu-id="e5a67-110">So a few Microsoft employees gathered up and started an effort to define a single, open card format and a set of SDKs that:</span></span>

- <span data-ttu-id="e5a67-111">упростили бы обмен карточками между ведущими приложениями;</span><span class="sxs-lookup"><span data-stu-id="e5a67-111">Would facilitate the interchange of cards between hosts</span></span>
- <span data-ttu-id="e5a67-112">позволили бы каждому ведущему приложению управлять стилем карточек, чтобы обеспечить согласованность оформления;</span><span class="sxs-lookup"><span data-stu-id="e5a67-112">Would allow each host to keep control over the styling of cards to ensure visual consistency</span></span>
- <span data-ttu-id="e5a67-113">упростили для ведущего приложения отображение карточек благодаря готовым к использованию пакетам SDK;</span><span class="sxs-lookup"><span data-stu-id="e5a67-113">Would make it easy for a host application to display cards with minimum effort via ready-to-use SDKs</span></span>
- <span data-ttu-id="e5a67-114">имели бы ценность для сторонних производителей и, в конечном итоге, стали бы широко применяться в отрасли.</span><span class="sxs-lookup"><span data-stu-id="e5a67-114">And that would also provide value to third parties and eventually get adopted widely by the industry</span></span>

## <a name="principles-governing-adaptive-cards"></a><span data-ttu-id="e5a67-115">Принципы, управляющие адаптивными карточками</span><span class="sxs-lookup"><span data-stu-id="e5a67-115">Principles governing Adaptive Cards</span></span>

1.  <span data-ttu-id="e5a67-116">**Адаптивная карточка — это _простой_ и _декларативный_ формат карточки.**</span><span class="sxs-lookup"><span data-stu-id="e5a67-116">**Adaptive Card is a _simple_ and _declarative_ card format**</span></span>

    1.  <span data-ttu-id="e5a67-117">Он не предназначен ни заменить HTML или XAML, ни стать их альтернативой.</span><span class="sxs-lookup"><span data-stu-id="e5a67-117">It is not meant as an HTML or XAML replacement/alternative</span></span>
    2.  <span data-ttu-id="e5a67-118">Для применения адаптивных карточек **не нужен код программной части**.</span><span class="sxs-lookup"><span data-stu-id="e5a67-118">There is **no "code behind"** with Adaptive Cards</span></span>
        1. <span data-ttu-id="e5a67-119">Разработчики карточек не могут внедрять пользовательский или произвольный код в свои полезные данные, поэтому ведущему приложению с адаптивными карточками никогда не требуется запускать сторонний код.</span><span class="sxs-lookup"><span data-stu-id="e5a67-119">Card authors cannot embed custom/arbitrary code with their payloads, and as a result an Adaptive Card host never needs to run third party code</span></span>
        2. <span data-ttu-id="e5a67-120">Динамичность и взаимодействие обеспечиваются исключительно с помощью декларативной разметки.</span><span class="sxs-lookup"><span data-stu-id="e5a67-120">Dynamism and interactivity are achieved solely via declarative markup</span></span>
    3.  <span data-ttu-id="e5a67-121">Это гарантирует, что усилия, необходимые для создания пакета SDK адаптивных карточек для новой платформы, останутся приемлемыми.</span><span class="sxs-lookup"><span data-stu-id="e5a67-121">This ensures that the effort necessary to build an Adaptive Card SDK for a new platform remains reasonable</span></span>

2.  <span data-ttu-id="e5a67-122">**Формат адаптивной карточки не может предписывать использование какой бы то ни было конкретной базовой технологии.**</span><span class="sxs-lookup"><span data-stu-id="e5a67-122">**The Adaptive Card format can't impose the use of any particular underlying technology**</span></span>

    1.  <span data-ttu-id="e5a67-123">Формат адаптивной карточки не зависит от JavaScript, C#, Python или любого другого языка.</span><span class="sxs-lookup"><span data-stu-id="e5a67-123">The Adaptive Card format does not rely on JavaScript, C#, Python or any other language</span></span>
    2.  <span data-ttu-id="e5a67-124">Аналогично, он не основывается на HTML или XAML, а также на любой другой платформе графического или пользовательского интерфейса.</span><span class="sxs-lookup"><span data-stu-id="e5a67-124">Similarly it doesn't rely on HTML or XAML or any other graphic/UI framework</span></span>
    3.  <span data-ttu-id="e5a67-125">Таким образом, адаптивные карточки можно отображать на любой платформе, если для нее существует отрисовщик.</span><span class="sxs-lookup"><span data-stu-id="e5a67-125">This way, Adaptive Cards can be rendered natively on any platform for as long as a renderer exists</span></span>

    1.  <span data-ttu-id="e5a67-127">Как и соответствующие пакеты SDK, этот формат должен предоставляться как открытый код, а его развитие должно осуществляться усилиями сообщества.</span><span class="sxs-lookup"><span data-stu-id="e5a67-127">Along with its SDKs, the format is to be open source and its evolution community driven</span></span>
    2.  <span data-ttu-id="e5a67-128">Таким образом, данный формат не принадлежит ни одной команде.</span><span class="sxs-lookup"><span data-stu-id="e5a67-128">The format is therefore not owned nor driven by any one team</span></span>

4.  <span data-ttu-id="e5a67-129">**Формат адаптивной карточки не предназначен только для решений корпорации Майкрософт.**</span><span class="sxs-lookup"><span data-stu-id="e5a67-129">**The Adaptive Card format is not designed "just for Microsoft's use"**</span></span>

    1.  <span data-ttu-id="e5a67-130">Вместо этого он разработан в соответствии с требованиями широкого спектра приложений и вариантов использования.</span><span class="sxs-lookup"><span data-stu-id="e5a67-130">Instead it is designed to suit the needs of a wide variety of applications and use cases</span></span>

    4.  <span data-ttu-id="e5a67-137">Для утверждения рабочей группой предлагаемые новые возможности:</span><span class="sxs-lookup"><span data-stu-id="e5a67-137">To be approved by the Working Group, proposed new features:</span></span>
